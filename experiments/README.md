# 实验代码

存放实验代码和脚本。每个实验建一个子目录。

## 命名规则

`exp-NNN-简短描述/`，NNN 从 001 开始，不复用编号。

## 子目录结构

```
experiments/
└── exp-001-描述/
    ├── log.md        # 实验说明、配置、命令、结果解释（使用 templates/experiment-log.md）
    ├── config.yaml   # 或等价配置记录
    └── ...           # 实验代码和脚本
```

## 规则

- 原始运行日志和大型输出不提交。
- 指标摘要写入 `log.md`，最终结果数据放入 `data/`。
- 每个实验在 `log.md` 中记录使用的数据集版本。
