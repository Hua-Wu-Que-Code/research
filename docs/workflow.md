# 科研工作流

本仓库遵循一条完整闭环：

```text
找文献
-> 整理论文
-> 构建领域知识库
-> 发现研究空白
-> 评估创新点
-> 做实验
-> 写作输出
-> 更新知识库和 TODO
```

## 1. 找文献

在 `literature/searches/` 中记录：

- 检索日期
- 检索来源
- 检索式
- 筛选条件
- 命中数量
- 筛选标准
- 覆盖不足或弱相关说明

使用 `templates/search-log.md`。

## 2. 整理论文

`literature/metadata/` 用于保存文献元数据、表格或文献管理工具导出文件。

`literature/paper-notes/` 用于保存结构化论文笔记。每篇论文笔记应包含：

- 文献元数据
- 研究问题
- 方法
- 数据集和指标
- 关键发现
- 局限性
- 与本项目的关系
- 可复用资源
- 核验状态

使用 `templates/paper-note.md`。

## 3. 构建领域知识库

知识库按概念组织，而不是按论文阅读顺序组织。

推荐页面：

- `wiki/problems/`：研究问题和任务定义
- `wiki/methods/`：方法族、核心假设和技术路线
- `wiki/datasets/`：数据集、数据构造、许可和限制
- `wiki/baselines/`：必须比较的基线和比较协议
- `wiki/metrics/`：指标定义和常见误用
- `wiki/failure-cases/`：已知弱点和负结果
- `wiki/gaps/`：尚未解决的研究空白

## 4. 发现研究空白

研究空白分析应回答：

- 已有工作已经解决了什么
- 还剩什么没有解决
- 为什么这个空白重要
- 哪个实验可以验证它
- 需要比较什么基线
- 需要什么数据和指标
- 审稿人可能质疑什么

使用 `templates/gap-analysis.md`。

## 5. 评估创新点

创新点评估使用 `templates/idea-review.md`，判断选一种：

- 可作为主贡献
- 只能作为组件
- 只能作为诊断信号
- 不值得继续

## 6. 做实验

每个实验都应在解释结果之前定义通过/失败标准。

每个实验目录应包含：

- 实验说明
- 配置文件或等价记录
- 运行脚本或命令记录
- 指标摘要
- 结果解释
- 实验决策

使用 `templates/experiment-log.md`。

## 7. 写作输出

`writing/` 用于保存论文、报告、返修回复和汇报材料。草稿中的主张应引用论文笔记、知识库页面和实验记录，而不是依赖没有证据支撑的记忆。

## 8. 更新项目状态

每次完成重要任务后：

- 更新相关知识库页面
- 更新 `TODO.md`
- 在 `meetings/` 中记录会议反馈
- 将未解决主张标为“待核验”
