# 领域知识库使用说明

这个目录用于构建“可持续增长”的领域知识库。

它采用三层结构：

1. **原始资料层 `raw/`**：保存不可变的来源材料、摘录、会议记录和资源索引。
2. **知识页层**：把资料整理成实体、概念、问题、方法、数据集、指标、研究空白、实验问题等互相链接的 Markdown 页面。
3. **规则层**：用 `SCHEMA.md`、`index.md`、`log.md` 约束格式、索引页面和记录每次更新。

## 每次使用前先读

1. `SCHEMA.md`：命名、元数据、标签、页面创建规则。
2. `index.md`：已有页面目录。
3. `log.md`：最近做过哪些更新。

## 推荐流程

```text
raw/ 放入来源材料
-> literature/ 形成论文笔记
-> wiki/ 形成概念页、方法页、问题页
-> gaps/ 形成候选研究空白
-> ideas/ 形成创新点评审
-> experiments/ 做实验验证
-> 再回写领域知识库和 TODO
```

## 重要原则

- 不把未核验内容写成事实。
- 不把 PDF、大数据、模型权重提交到 Git。
- 重要判断必须能追溯到论文笔记、原始资料或实验记录。
- Wiki 页面之间使用 `[[wikilinks]]` 交叉引用。
- 每次创建或更新页面后，同步更新 `index.md` 和 `log.md`。
